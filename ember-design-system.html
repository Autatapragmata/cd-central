01 — PRINCIPLES

Five ideas that guide every pixel.

Design systems are only as good as the principles behind them. These are ours.

P.01

Clarity over cleverness

If a label, layout, or interaction needs an explanation, rework it. The best interfaces feel obvious in retrospect.

P.02

Restraint with color

Neutrals do 90% of the work. Accent color is reserved for moments that genuinely deserve attention — primary actions, active states, critical alerts.

P.03

Type is the interface

Hierarchy, rhythm, and readability are set through typography first. Decoration follows; it never leads.

P.04

Motion with purpose

Every animation answers a question: where did this come from, where did it go, what changed? No decorative loops, no attention grabs.

P.05

Accessible by default

Contrast, focus states, motion preferences, and keyboard paths are not features. They're entry conditions.

P.06

Consistency compounds

Reuse tokens, reuse components, reuse patterns. The system is a promise to the user that the next screen will behave like the last.

06 — MOTION

Fast, soft, and never in the way.

Three durations. Three curves. Motion acknowledges cause and effect — nothing more. Respect prefers-reduced-motion.

Ease-out · 120ms

Hover to preview. For entrances, reveals, expanding surfaces.

Ease-in-out · 240ms

Hover to preview. For transitions between two defined states.

Linear · 480ms

Hover to preview. For progress, loading, continuous motion only.

07 — APPLICATION

How to use the system well.

A handful of habits that keep Ember feeling like Ember across teams and quarters.

DO

Use Signal for a single primary action per view.

One decision is always clearer than two. Secondary actions take the ink or ghost treatment.

DON'T

Paint sections or backgrounds with Signal.

The accent loses its power at scale. Use it on components, not containers.

DO

Pair Fraunces with generous space.

Display type is already loud. Let it breathe — tight margins mute its effect.

DON'T

Mix Fraunces and Geist within a heading.

Switch fonts across hierarchy levels, never within a single line. The break should feel deliberate.

DO

Snap every spacing value to the scale.

If you find yourself typing 14px or 22px, go up or down to the next token. Consistency is invisible and cumulative.

DON'T

Animate content that hasn't changed.

Decorative motion erodes trust. If nothing updated, nothing should move.
